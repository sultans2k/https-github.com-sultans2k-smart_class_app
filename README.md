## Overview

**Smart Class** is a Flutter-based mobile application designed for teachers and students. It records full classroom sessions as raw audio, then — after the session ends — processes the recording into a complete text transcript and a structured AI-generated summary, cross-referenced against the course's uploaded PDF materials.

> No real-time processing. No internet dependency during recording. Just capture now, study smarter later.

---

## How It Works

```
Teacher records lecture
        │
        ▼
 Raw audio saved locally
 (no live processing)
        │
        ▼
 Teacher ends session
        │
        ▼
 Audio uploaded for transcription
 (Whisper / Speech-to-Text)
        │
        ▼
 Transcript compared against
 uploaded PDF curriculum files
        │
        ▼
 AI generates structured summary
 + full transcript
        │
        ▼
 Archived & ready for students
```

The post-recording approach significantly improves transcription accuracy compared to live processing, since the full audio context allows better language correction and speed handling.

---

## Features

- **Full Session Recording** — Record entire lectures as raw audio with no performance overhead
- **Batch Transcription** — Upload and transcribe the full audio in one pass for higher accuracy
- **AI-Powered Summaries** — Structured summaries generated by Gemini AI
- **Curriculum Cross-referencing** — AI compares the teacher's explanation against PDF course materials
- **Session Archive** — All sessions stored and accessible anytime; content is ready after processing
- **Chat Interface** — Ask questions about any session using the built-in AI chat
- **Cross-Platform** — Runs on Android and iOS

## Getting Started

### Prerequisites

- [Flutter SDK](https://docs.flutter.dev/get-started/install) 3.x+
- Dart 3.x
- A Google Gemini API key — [Get one here](https://ai.google.dev)

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/Hatim2004/smart_class_app.git
cd smart_class_app/my_app

# 2. Install dependencies
flutter pub get

# 3. Configure API keys (see below)

# 4. Run the app
flutter run
```

### API Key Configuration

> **Never hardcode API keys in source files.**

Create a `.env` file in `my_app/` (already in `.gitignore`):

```env
GEMINI_API_KEY=your_gemini_key_here
OPENAI_API_KEY=your_openai_key_here
ELEVENLABS_API_KEY=your_elevenlabs_key_here
```

Then load them in your app using [`flutter_dotenv`](https://pub.dev/packages/flutter_dotenv).
